Running legacy mod_perl code with Plack

Peter Makholm, one.com

10 years ago...

mod_perl is great!

... no per-request start up time

... some kind of persistency

... hook into Apache internals

mod_perl, The downside

Some black magic included

Obscurities like read-only @INC

Can be hard to debug complex client/server interactions

New kid on the block

Plack/PSGI is awesome!

... not tied to Apache

... no hidded magic

... ease of debugging, profiling, testing

PSGI, The basics

An application is a code ref

... hashref ($env) with all request information as argument

... return status code, headers, and body

PSGI, The building blocks

Multiple server backends: mod_perl, FastCGI, pure perl, ...

Middelware, wraps around your application

The Plack::Builder DSL for configuring your application

The idea

Use PSGI's $env to fake mod_perl's $req

Prior art:

... Apache::FakeRequest, Apache2::FakeRequest

... lot of testing code

The implementation

Available as Plack::App::FakeApache

... Implements the parts of $req I need

... Used as development tool, not for deployment

... patches and wishes welcome

Batteries included

Profit in three simple steps

1. Edit a single line in examples/generic.psgi

2. $ plackup examples/generic.psgi

3. Go to http://localhost:5000/
